# 1. 그리디

#### 이론
 - 탐욕적 뜻: 현재 상황에서 지금 당장 좋은 것만 고르는 방법을 의미
 - 문제 풀이를 위한 최소한의 아이디어를 떠올리고 이것이 정당한지 검토할 수 있어야 답을 도출 할 수 있음

#### 문제 풀때 앞에 숫자, 배열 등을 받는 방법
 -  N, M, K를 공백으로 구분하여 입력받기
n, m, k = map(int, input().split())
 -  N개의 수를 공백으로 구분하여 입력받기
data = list(map(int, input().split()))


# 2. 구현

 - '구현'이란 머릿속에 있는 알고리즘을 소스코드로 바꾸는 과정, 
   모든 범위의 코딩테스트 문제 유형을 포함하는 개념
 - 책에서는 완전 탐색과 시뮬레이션 유형 모두 구현으로 다루고 있음
 - 완전 탐색: 모든 경우의 수를 주저 없이 다 계산하는 해결 방법
 - 시뮬레이션: 문제에서 제시한 알고리즘을 한 단계씩 차례대로 직접 수행해야 하는 문제 유형

 - (알고리즘 문제를 풀 때는) 시간 제한과 데이터의 개수를 먼저 확인한 뒤에
   이 문제를 어느 정도의 시간 복잡도의 알고리즘으로 작성해야 풀 수 있을 것인지 예측할 수 있어야 함

#### 문제 참고
 - '상하좌우' 문제에서는 dx, dy 리스트를 선언하여 이동할 방향을 기록,
   '왕실의 나이트' 문제처럼 steps 변수가 dx, dy 변수의 기능을 대신하여 수행하는 경우도 있음
